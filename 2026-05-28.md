## 1050. Actors and Directors Who Cooperated At Least Three Times

🔗 https://leetcode.com/problems/actors-and-directors-who-cooperated-at-least-three-times/

### 解法

```sql
SELECT actor_id, director_id
FROM ActorDirector
GROUP BY actor_id, director_id
HAVING COUNT(*) >= 3;
```

### 重點

- `GROUP BY` 可以同時以多個欄位分組，這裡是「同一個演員 + 同一個導演」為一組。
- 分組之後用 `HAVING COUNT(*) >= 3` 篩選合作至少 3 次的組合。

## 1068. Product Sales Analysis I

🔗 https://leetcode.com/problems/product-sales-analysis-i/

### 解法

```sql
SELECT p.product_name, s.year, s.price
FROM Sales AS s
LEFT JOIN Product p ON s.product_id = p.product_id;
```

### 重點

- 基本的 JOIN 題，從 Sales 出發，把 Product 的 product_name 接進來。
- 用 `LEFT JOIN` 保留所有銷售記錄，即使沒有對應的產品資料也不會漏掉。

## 1075. Project Employees I

🔗 https://leetcode.com/problems/project-employees-i/

### 解法

```sql
SELECT p.project_id, ROUND(AVG(e.experience_years), 2) AS average_years
FROM Project p
LEFT JOIN Employee e ON p.employee_id = e.employee_id
GROUP BY project_id;
```

### 錯誤紀錄

❌ 第一次忘了加 `ROUND()`，結果小數位數太多，沒有四捨五入到小數點後兩位。

### 重點

- `AVG()` 會算出平均值，但小數位數不固定，題目要求保留兩位小數要用 `ROUND(值, 位數)`。
- `ROUND(AVG(e.experience_years), 2)` — 先算平均，再四捨五入到第 2 位小數。

> 之後只要題目說「保留幾位小數」或「四捨五入」，就記得包一層 `ROUND()`。

## 1084. Sales Analysis III

🔗 https://leetcode.com/problems/sales-analysis-iii/

### 解法

```sql
SELECT p.product_id, p.product_name
FROM Product AS p
LEFT JOIN Sales s ON p.product_id = s.product_id
GROUP BY p.product_id, p.product_name
HAVING MIN(s.sale_date) >= '2019-01-01'
    AND MAX(s.sale_date) <= '2019-03-31';
```

### 重點

- 題目要找「所有銷售都在 2019 Q1 以內」的產品，用 `MIN` 和 `MAX` 來確認日期範圍。
  - `MIN(sale_date) >= '2019-01-01'`：最早的銷售不早於 Q1 開始
  - `MAX(sale_date) <= '2019-03-31'`：最晚的銷售不超過 Q1 結束
- 如果有任何一筆銷售在範圍外，MIN 或 MAX 就會超出限制，這筆產品就會被過濾掉。
- 用 `MIN/MAX` 來驗證「所有資料都在範圍內」是一個很好用的技巧。

## 今日學習重點總結

| 概念 | 說明 | 範例題 |
|------|------|--------|
| `GROUP BY` 多欄位 | 可以同時用多個欄位做分組 | 1050, 1084 |
| `ROUND(值, 位數)` | 四捨五入到指定小數位數 | 1075 |
| `MIN/MAX` 驗證範圍 | 確認群組內所有資料都在某個範圍內 | 1084 |

### ROUND 語法

```sql
ROUND(AVG(欄位), 2)   -- 平均後四捨五入到小數第 2 位
ROUND(SUM(欄位), 0)   -- 加總後四捨五入到整數
```

### 用 MIN/MAX 確認「全部都在範圍內」

```sql
-- 確認這個群組的所有日期都在 2019 Q1 內
HAVING MIN(sale_date) >= '2019-01-01'
   AND MAX(sale_date) <= '2019-03-31'
```
