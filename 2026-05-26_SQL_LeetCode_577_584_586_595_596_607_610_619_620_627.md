## 577. Employee Bonus

🔗 https://leetcode.com/problems/employee-bonus/

### 解法

```sql
SELECT 
    e.name,
    b.bonus
FROM Employee e
LEFT JOIN Bonus b
    ON b.empId = e.empId
WHERE b.bonus < 1000 OR b.bonus IS NULL;
```

### 重點

- 用 `LEFT JOIN` 保留所有員工，沒有 bonus 記錄的人 `b.bonus` 會是 NULL。
- `WHERE` 條件要同時處理兩種情況：bonus 小於 1000，**或者** 根本沒有 bonus（IS NULL）。
- 如果只寫 `b.bonus < 1000`，沒有 bonus 的人會被過濾掉，因為 `NULL < 1000` 在 SQL 中不成立。

## 584. Find Customer Referee

🔗 https://leetcode.com/problems/find-customer-referee/

### 解法

```sql
SELECT name
FROM Customer
WHERE referee_id != 2 OR referee_id IS NULL;
```

### 重點

- 跟上一題一樣的邏輯：`referee_id != 2` 不會把 NULL 的人包含進來，要另外加 `OR IS NULL`。
- SQL 中 NULL 跟任何值做比較（包括 `!=`）結果都不是 TRUE，所以 NULL 的列會直接被排除。

> 這兩題提醒我：只要欄位可能是 NULL，條件就要記得補上 `OR ... IS NULL`。

## 586. Customer Placing the Largest Number of Orders

🔗 https://leetcode.com/problems/customer-placing-the-largest-number-of-orders/

### 解法

```sql
SELECT customer_number
FROM Orders
GROUP BY customer_number
ORDER BY COUNT(*) DESC
LIMIT 1;
```

### 重點

- `GROUP BY` 之後用 `ORDER BY COUNT(*) DESC` 把訂單數最多的排到第一。
- `LIMIT 1` 只取第一筆，就是訂單最多的顧客。
- 這個方法比用子查詢 `MAX()` 更簡潔。

## 595. Big Countries

🔗 https://leetcode.com/problems/big-countries/

### 解法

```sql
SELECT 
    name,
    population,
    area
FROM World
WHERE area >= 3000000 OR population >= 25000000;
```

### 錯誤紀錄

❌ 第一次用了 `>` 而不是 `>=`，導致剛好等於門檻的國家被排除。

### 重點

- 題目說「至少 300 萬平方公里或 2500 萬人口」，「至少」就是 `>=`，不是 `>`。
- 看到「至少/以上/不少於」→ `>=`；「超過/大於」→ `>`。

## 596. Classes More Than 5 Students

🔗 https://leetcode.com/problems/classes-more-than-5-students/

### 解法

```sql
SELECT class
FROM Courses
GROUP BY class
HAVING COUNT(class) >= 5;
```

### 重點

- 標準的 `GROUP BY` + `HAVING` 用法，跟 182 題（重複 email）邏輯一樣。
- `HAVING COUNT(...) >= 5`：篩選出學生數達到 5 人以上的班級。

## 607. Sales Person

🔗 https://leetcode.com/problems/sales-person/

### 解法

```sql
SELECT s.name
FROM SalesPerson s
WHERE s.sales_id NOT IN (
    SELECT o.sales_id
    FROM Orders o
    JOIN Company c ON o.com_id = c.com_id
    WHERE c.name = 'RED'
);
```

### 重點

- 用 `NOT IN` + 子查詢：先找出「有跟 RED 公司做生意的業務員 id」，再排除他們。
- 子查詢先篩出有問題的人，主查詢再用 `NOT IN` 反向取得乾淨的結果，這個邏輯很通用。
- 也可以用 `LEFT JOIN` + `IS NULL` 做，但這裡子查詢更直觀。

## 610. Triangle Judgement

🔗 https://leetcode.com/problems/triangle-judgement/

### 解法

```sql
SELECT 
    x, y, z,
    CASE 
        WHEN x + y > z AND x + z > y AND y + z > x THEN 'Yes'
        ELSE 'No'
    END AS triangle
FROM Triangle;
```

### 重點

- `CASE WHEN ... THEN ... ELSE ... END`：SQL 的條件判斷，等同於程式語言的 if/else。
- 三角形條件：任意兩邊之和大於第三邊，三個條件都要成立（用 `AND` 連接）。
- `CASE WHEN` 可以直接放在 `SELECT` 裡，生成一個新的計算欄位。

## 619. Biggest Single Number

🔗 https://leetcode.com/problems/biggest-single-number/

### 解法

```sql
SELECT MAX(num) AS num
FROM (
    SELECT num
    FROM MyNumbers
    GROUP BY num
    HAVING COUNT(*) = 1
) AS single_numbers;
```

### 重點

- 兩層查詢：內層先找出「只出現一次」的數字，外層再用 `MAX()` 取最大值。
- 如果沒有只出現一次的數字，`MAX()` 會回傳 NULL，剛好符合題目要求。
- 子查詢要記得給 alias（這裡是 `single_numbers`），不然 MySQL 會報錯。

## 620. Not Boring Movies

🔗 https://leetcode.com/problems/not-boring-movies/

### 解法

```sql
SELECT *
FROM Cinema
WHERE MOD(id, 2) = 1 AND description != 'boring'
ORDER BY rating DESC;
```

### 重點

- `MOD(id, 2) = 1`：取餘數為 1，即奇數 id。也可以寫成 `id % 2 = 1`。
- 兩個條件同時成立用 `AND`。
- 最後 `ORDER BY rating DESC` 按評分由高到低排列。

## 627. Swap Salary

🔗 https://leetcode.com/problems/swap-salary/

### 解法

```sql
UPDATE Salary
SET sex = CASE
    WHEN sex = 'f' THEN 'm'
    ELSE 'f'
END;
```

### 重點

- 這題用的是 `UPDATE`，不是 `SELECT`，直接修改資料。
- `CASE WHEN` 放在 `SET` 裡面：f 改 m、m 改 f，一次完成互換。
- 不需要 `WHERE`，因為全部的列都要更新。

## 今日學習重點總結

### 新概念整理

| 概念 | 說明 | 範例題 |
|------|------|--------|
| `OR ... IS NULL` | 欄位可能為 NULL 時，條件要額外處理 | 577, 584 |
| `ORDER BY ... LIMIT` | 排序後取第幾筆，常用來找最大/最小 | 586 |
| `>= vs >` | 「至少/以上」是 `>=`，「超過」才是 `>` | 595 |
| `NOT IN` + 子查詢 | 先找出「排除的對象」再反向篩選 | 607 |
| `CASE WHEN` | SQL 的條件判斷，可放在 SELECT 或 SET 裡 | 610, 627 |
| `MOD(n, 2)` | 取餘數，判斷奇偶數 | 620 |
| `UPDATE` + `CASE WHEN` | 根據條件更新不同的值 | 627 |

### NULL 相關的坑（容易忘）

```
x != 2          → NULL 的列不會被包含（NULL != 2 不是 TRUE）
x != 2 OR x IS NULL  → NULL 的列才會被包含
```

### CASE WHEN 語法

```sql
CASE
    WHEN 條件1 THEN 結果1
    WHEN 條件2 THEN 結果2
    ELSE 預設結果
END
```
